# Panda Redesign Study

Huashu-design prototype for a more modern, graphical, and sophisticated Panda
visual system.

## Open

Open `index.html` in a browser, or serve this directory with any static server.

## Contents

- `Brief`: assumptions, source tokens, and redesign constraints.
- `Directions`: three visual routes with the recommended synthesis.
- `Landing`: public marketing direction.
- `Projects`: authenticated project switcher.
- `Workbench`: main operational shell.
- `States`: permission, changed work, and recoverable checkpoint state.
- `Mobile`: Session, Chat, and Proof focus views.
- `Settings`: General, Providers, Automation, Advanced.
- `Admin`: Dashboard, Users, Analytics, System, Security.
- `Share`: public redacted shared-thread artifact.

## Direction

Recommended route: `C / Recovery Console`.

It keeps Panda's existing warm brutalist foundation while making the product
feel more graphical through tighter composition, stronger panel hierarchy,
denser proof surfaces, and a clearer owner-vs-public trust boundary.
